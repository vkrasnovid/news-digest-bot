# Fix code review issues: SEC-1, TG-1, CQ-1

**Date:** 2026-03-20 11:00
**Files:** index.html
**Severity:** medium

## Problem

Three medium-severity issues identified in CODE_REVIEW.md:

1. **SEC-1** — `state.lastSave` was not sanitized in `loadState()`. A future timestamp in localStorage caused `Date.now() - state.lastSave` to be negative, which silently deducted coins on app startup.
2. **TG-1** — `tg.sendData()` was used as a fallback in `shareScore()`. This API closes the Mini App immediately, destroying the user's game session mid-play.
3. **CQ-1** — `updateUI()` called `renderShop()` (full `innerHTML` wipe + DOM rebuild) whenever a card transitioned from affordable to unaffordable. Since `updateUI()` runs on every tap and every auto-tick, this caused a full rebuild every second when coin balance oscillated near an upgrade threshold — visible flicker and GC pressure.

## Root Cause

1. **SEC-1** — The numeric-clamping block added in a previous fix covered all gameplay-critical fields but omitted `lastSave`. `calcOfflineEarnings()` did not guard against negative elapsed time.
2. **TG-1** — `sendData()` was used as a reasonable-seeming Telegram fallback, but its side effect (closing the WebApp) was overlooked.
3. **CQ-1** — The original shop affordability check mixed two concerns: lightweight class toggling (can-buy → disable) and full DOM rebuild (disable → can-buy). The rebuild path was left in the else-branch as a shortcut to also reset listeners. Separating listener management from affordability toggling was not done.

## Solution

1. **SEC-1** — `state.lastSave = Math.min(Date.now(), Math.max(0, Number(state.lastSave) || 0))` in `loadState()`. Also wrapped `calcOfflineEarnings()` elapsed with `Math.max(0, ...)` as a defence-in-depth guard.
2. **TG-1** — Removed the `else if (tg?.sendData)` branch entirely. The browser fallback (`navigator.share` → `navigator.clipboard`) already handles this case safely.
3. **CQ-1** — Added `card.dataset.id = upg.id` in `renderShop()`, removed per-card click/touchend listeners there, replaced `renderShop()` in `updateUI()` with `card.classList.add('disabled')`, and registered a single delegated listener on `#shop-panel` in `DOMContentLoaded`.

## Prevention

- When adding numeric-clamping blocks for saved state, audit ALL fields that feed into arithmetic — especially timestamps used in elapsed-time calculations.
- Before using any Telegram WebApp API method, check its documented side effects (`sendData` closes the app; this is a footgun for mid-session fallbacks).
- For UI components driven by high-frequency ticks, keep the per-tick path to class toggles only. Attach event listeners once (delegation or one-time registration), never inside the tick loop.
